# 01-需求分析输入.md

---

## 一、项目概述

**项目名称：** 学院工作任务栏系统

**项目背景：**
根据学院重点工作计划，当前使用纸质记录或Excel表格方式管理工作计划，存在信息分散、状态不透明、协同困难等问题，需要开发一套系统化的工作任务展示与管理工具。

**项目目标：**
1. 集中管理学院各个老师负责的项目和工作任务
2. 实时展示各项任务的完成状态和进度
3. 支持按负责人、周期、分类等多维度查看任务
4. 提升任务管理的透明度和协同效率
5. 支持个人待办事项管理，方便用户规划日常工作
6. 实现会议管理及实时提醒，避免错过重要会议

**预期效果：**
1. 各个老师可以看到自己当前有哪些项目，其中完成了哪些，未完成的又有哪些
2. 管理员可以统一管理所有任务，及时跟踪进度
3. 领导层可以快速了解学院整体工作进展
4. 支持按**教学周**（如第6周、第8周）筛选任务；并支持按**学期**筛选（学期与周通过周期主数据关联，先选学期再选周或列表按学期过滤）
5. 用户可为自己创建待办清单，提高工作效率
6. 会议开始前按预设提前量自动提醒，确保准时参会

**视图与数据关系（说明）：**
- **全局任务列表**与**我的项目视图**使用同一套任务数据；后者仅默认筛选「当前用户为负责人」的任务，筛选条件与列表列展示与全局列表一致。
- **任务分类**：系统维护分类主数据（名称、排序、展示色等）；创建/筛选任务时从主数据中选择，不在界面上随意输入未定义分类。
- **协助人**：非独立登录角色；某用户被设为任务的协助人时，在该任务范围内享有 A-004 所列权限。

## 二、功能需求汇总

### 2.1 核心功能模块

| 模块 | 功能数量 | 优先级 |
|------|----------|--------|
| 任务管理模块 | 9项 | 高 |
| 个人待办模块 | 6项 | 高 |
| 会议管理模块 | 8项 | 高 |
| 账号与权限模块 | 4项 | 高 |
| 通知提醒模块 | 2项 | 中 |
| 周期管理模块 | 3项 | 中 |
| 数据导入导出 | 2项 | 低 |

### 2.2 功能清单

#### 任务管理模块

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|----------|----------|----------|--------|
| T-001 | 任务列表展示 | 首页以列表形式展示任务，显示任务名称、负责人、截止日期、状态、进度等关键信息；领导/管理员默认看全院，教师默认看本人负责及协助的任务（见权限矩阵） | 高 |
| T-002 | 任务分类筛选 | 按任务分类筛选；分类取值来自系统分类主数据，预设示例：教学工作、科研工作、党建工作、行政工作、学生工作、产业学院工作等（可由管理员扩展） | 高 |
| T-003 | 负责人筛选 | 支持按负责人筛选，快速查看某个老师的所有任务 | 高 |
| T-004 | 周期与学期筛选 | 支持按教学周（周期）筛选任务列表；支持按学期筛选（学期字段归属周期主数据）。交互上可提供「学期 → 周」两级过滤或等价组合条件 | 高 |
| T-005 | 任务搜索 | 支持按任务名称关键词搜索，快速定位任务 | 中 |
| T-006 | 状态筛选 | 支持按任务状态筛选：全部、待开始、进行中、已完成、已延期、已取消（界面文案与存储取值对应关系见 §2.4） | 中 |
| T-007 | 进度展示 | 直观展示任务完成百分比，支持进度条可视化 | 高 |
| T-008 | 新增任务 | 管理员可新增任务：任务名称、负责人、协助人、所属周期、开始/截止日期、分类、优先级等 | 高 |
| T-009 | 编辑/删除任务 | 管理员可编辑、删除任务；教师不可删除任务本体，可在授权范围内更新进度（见权限矩阵与 A-004） | 中 |

#### 个人待办模块

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|----------|----------|----------|--------|
| TD-001 | 我的项目视图 | 用户登录后可查看自己为**负责人**的任务集合，支持按状态、周期、学期等维度筛选（与 T-004 维度一致） | 高 |
| TD-002 | 待办清单 | 用户可为自己负责的任务添加子待办事项，形成可检查的具体步骤；也支持不关联任务的纯个人待办（task 为空） | 高 |
| TD-003 | 待办勾选 | 用户对待办事项进行勾选完成操作，完成后打勾并划线显示 | 高 |
| TD-004 | 待办优先级 | 待办支持优先级标记（紧急、重要、一般、较低），用于排序与展示，与数据库优先级枚举一致 | 中 |
| TD-005 | 截止时间 | 待办事项可设置个人截止时间；到期前在列表中高亮或提醒（与 N-001 范围协调：至少任务截止有全局规则，待办截止以列表提醒为主） | 中 |
| TD-006 | 共享待办 | 可将待办共享给其他人员；共享时可指定权限为仅查看或可编辑，接收方在「分享给我的」列表中查看 | 低 |

#### 会议管理模块

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|----------|----------|----------|--------|
| M-001 | 会议列表 | 展示会议，支持按日期、状态筛选 | 高 |
| M-002 | 新增会议 | **本期仅管理员**可创建会议，填写会议名称、时间、地点、参会人员、会议议题等 | 高 |
| M-003 | 编辑/删除会议 | 管理员可修改会议信息、取消会议 | 中 |
| M-004 | 会议提醒 | 每条会议可配置提前提醒时间（如开始前 15 分钟 / 30 分钟 / 1 小时 / 1 天等）；**默认 30 分钟**。到达设定时间前触发提醒，支持浏览器通知与系统提示音（在用户已授权前提下） | 高 |
| M-005 | 参会确认 | 参会人员可确认是否出席 | 中 |
| M-006 | 会议日历 | 以日历视图展示会议安排 | 中 |
| M-007 | 历史会议 | 归档已结束会议，支持查看历史记录 | 低 |
| M-008 | 会议通知权限 | 首次使用会议提醒前引导用户授权浏览器通知权限 | 高 |

#### 通知提醒模块

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|----------|----------|----------|--------|
| N-001 | 截止提醒 | 任务截止日期前 1 天在列表或详情中显著标红/角标提醒（不强制单独弹窗，可与列表样式联动） | 中 |
| N-002 | 状态变更审计 | 任务关键字段（如状态、进度）变更写入操作审计日志，供管理员或运维追溯；**本期不向普通用户推送「通知消息」**，与即时通讯式通知区分 | 低 |

#### 账号与权限模块

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|----------|----------|----------|--------|
| A-001 | 账号管理 | 管理员统一创建账号，不开放用户自主注册 | 高 |
| A-002 | 密码安全 | 密码加密存储，登录失败锁定机制 | 高 |
| A-003 | 人员管理 | 人员信息 CRUD，新入职/调离处理 | 高 |
| A-004 | 协助人权限 | 被指定为任务协助人的用户：可查看该任务、更新进度、为该任务添加/维护待办；不可删除任务、不可改负责人等核心属性 | 高 |

#### 周期管理模块

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|----------|----------|----------|--------|
| P-001 | 周期列表 | 管理员维护教学周周期：名称、起止日期、所属学期、排序 | 中 |
| P-002 | 周期维护 | 管理员新增、编辑、删除周期；删除前需业务校验（如是否仍有关联任务） | 中 |
| P-003 | 周期任务复制 | 管理员可将某一周期下的任务批量复制到目标周期（与开发与 API 设计一致），用于新学期/新周次初始化 | 中 |

#### 数据导入导出模块

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|----------|----------|----------|--------|
| D-001 | Excel 导入任务 | 管理员使用标准模板批量导入任务；需校验必填字段、周期/负责人/分类存在性；失败行可提示原因 | 低 |
| D-002 | Excel 导出任务 | 管理员与领导层可按筛选条件导出任务列表（字段范围与权限一致）；教师无导出权限 | 低 |

### 2.3 术语与界面-存储对照（任务状态）

| 界面文案 | 含义说明 |
|----------|----------|
| 待开始 | 未开工 |
| 进行中 | 正在执行 |
| 已完成 | 正常完结 |
| 已延期 | 超期未完成或标记延期 |
| 已取消 | 任务作废，不再参与进度统计 |

（具体存储枚举名由设计/数据库约定，测试用例以本表语义为准。）

## 三、用户角色分析

### 3.1 角色定义

| 角色 | 人数 | 说明 |
|------|------|--------|
| 管理员 | 1-2人 | 学院行政人员/秘书，负责系统运维、任务与会议维护、周期与分类主数据 |
| 教师/负责人 | 10-50人 | 学院任课老师、项目负责人；可被加为任务协助人 |
| 领导层 | 3-5人 | 院长/副院长/系主任 |

### 3.2 角色权限矩阵

| 功能 | 管理员 | 教师 | 领导层 |
|------|--------|------|--------|
| 任务：创建/删除/改核心属性 | ✓ | ✗ | ✗ |
| 任务：查看 | ✓ 全部 | ✓ 本人负责或协助的任务 | ✓ 全部（只读） |
| 任务：更新进度/状态（授权范围内） | ✓ | ✓ 本人负责或协助的任务 | ✗ |
| 会议：创建/编辑/取消 | ✓ | ✗（本期） | ✗ |
| 会议：查看与参会确认 | ✓ | ✓ | ✓ |
| 待办：本人及共享 | ✓ 全部待办（运维） | ✓ 本人执行/创建及被共享的待办 | ✓ 关联任务的待办 + 全院任务维度统计；**不可查看他人无任务关联的纯个人待办** |
| 周期与分类主数据 | ✓ | ✗ | ✗ |
| 数据导入/导出 | ✓ | ✗ | ✓ 仅导出，与 D-002 一致 |
| 人员与账号管理 | ✓ | ✗ | ✗ |

**说明：** 本期不设独立「系统设置」菜单；运维相关能力拆分为周期与分类维护、账号与人员管理、手动备份（见非功能需求）等具体功能。若后续合并为设置页，再单独立项。

## 四、非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 系统响应时间 | 页面加载 ≤ 2秒，操作响应 ≤ 500ms |
| 并发用户数 | 支持20用户同时在线 |
| 数据容量 | 支持1000+条任务、500+条会议记录 |

### 4.2 安全需求

| 项目 | 要求 |
|------|------|
| 用户认证 | 账号密码登录 |
| 密码存储 | bcryptjs加密存储 |
| 登录锁定 | 连续5次失败锁定30分钟 |
| Session有效期 | 24小时 |

### 4.3 可用性需求

| 项目 | 要求 |
|------|------|
| 系统可用性 | 99%以上 |
| 提醒准确性 | 会议提醒误差不超过1分钟 |
| 数据备份 | 支持手动备份 |

## 五、约束条件

| 约束类型 | 约束内容 |
|----------|----------|
| 技术约束 | 采用Vue.js + Node.js + MySQL技术栈 |
| 部署约束 | 部署在学院本地服务器 |
| 浏览器约束 | 支持Chrome、Firefox、Edge最新版本 |

## 六、验收标准

### 6.1 功能验收

| 模块 | 验收条件 |
|------|----------|
| 任务管理 | 任务 CRUD（权限符合矩阵）、筛选（含周期/学期/分类）、搜索、状态含已取消、导出符合角色 |
| 个人待办 | 待办增删改查、勾选完成、关联任务与纯个人待办、共享与权限级别 |
| 会议管理 | 会议 CRUD（创建仅限管理员）、日历视图、可配置提前提醒、浏览器通知链路 |
| 周期管理 | 周期主数据维护、任务与周期关联正确 |
| 账号权限 | 登录登出、锁定、Session、人员与账号管理、协助人行为符合 A-004 |
| 数据导入导出 | Excel 模板导入、导出字段与权限符合 D-001/D-002 |
| 审计 | 关键任务变更可追溯（N-002） |

### 6.2 界面验收

| 验收项 | 标准 |
|--------|------|
| 界面风格 | 简约现代、扁平化设计 |
| 动画效果 | 流畅、自然、无卡顿 |
| 布局与分辨率 | 在约定桌面分辨率（≥1024px 宽）下布局正常、无错位；不要求适配手机端 |
