### 2.2 技术架构分层

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 表现层 | Vue.js 3 + Element Plus | 组件化开发，响应式UI |
| 路由层 | Vue-Router 4 | SPA路由管理 |
| 状态层 | Pinia | Vue3官方推荐状态管理 |
| 请求层 | Axios | HTTP请求封装 |
| 服务端 | Node.js + Express | 轻量级后端服务 |
| 数据库 | MySQL 8.0 | 关系型数据库 |
| 定时任务 | node-cron | 会议提醒定时器 |
| 会话管理 | express-session | Session存储 |

## 五、状态管理设计

### 5.1 Pinia Store 模块划分

| Store | 职责 | 主要状态 |
|-------|------|----------|
| userStore | 用户信息管理 | userInfo, isLoggedIn |
| taskStore | 任务数据管理 | taskList, filters, pagination |
| meetingStore | 会议数据管理 | meetingList, calendarEvents |
| todoStore | 待办数据管理 | todoList, groupedTodos |
| appStore | 全局应用状态 | sidebarOpened, loading |

### 5.2 状态数据流

```
┌─────────────┐     dispatch      ┌─────────────┐     actions      ┌─────────────┐
│   View      │ ───────────────▶ │   Store     │ ───────────────▶ │    API     │
│  组件       │                  │   (Pinia)   │                  │   服务      │
└─────────────┘                  └─────────────┘                  └─────────────┘
       ▲                                 │                                │
       │                                 │                                │
       │                                 ▼                                │
       │                         ┌─────────────┐                        │
       └──────────────────────── │   State     │ ◀────────────────────────┘
                                 │   状态更新   │
                                 └─────────────┘
```

## 八、扩展性设计

### 8.1 预留扩展接口

| 扩展点 | 扩展方向 |
|--------|----------|
| 消息通知 | 支持邮件通知、短信通知 |
| 数据导入 | 支持更多文件格式 |
| 统计分析 | 任务完成率报表、甘特图 |
| 移动端 | 响应式适配或小程序 |

### 8.2 模块解耦

- 前端组件按功能模块拆分，独立开发和测试
- 后端服务层与控制器分离，便于业务扩展
- 数据库设计预留扩展字段
